\section{Evaluation: ModuleComposeBench}

ModuleComposeBench evaluates integration of existing RTL/IP modules: wrapper generation, stream chaining, bus attachment, width adaptation, protocol bridging, CDC adapter insertion, and unsafe ambiguity rejection. Metrics include first-pass compose success, lint/elaboration, simulation, bounded formal smoke, unsafe rejection, JSON AST equivalence, QoR delta, and repair turns.

The current artifact answers five evidence-bounded questions: whether intended compositions are accepted, unsafe compositions are rejected, accepted wrappers pass open-source RTL validation, selected ready/valid and width properties survive bounded formal checks, and generated wrappers remain comparable with hand-written references.

For the tested profiles, the schema-guided MICO JSON AST repair baseline reaches
36/36 positive compiler and lint passes on the public-development split and
10/10 on the held-out split in every profile. It also rejects 26/26 public unsafe
tasks and 10/10 held-out unsafe tasks in every profile. Direct Verilog and
SystemVerilog-interface prompts never reach positive lint pass rates comparable
to the structured baselines: their best rows are 8/36 public positives and 2/10
held-out positives. The MICO-source baseline produces MICO-looking text but has
0 positive compiler passes in the matrix, illustrating that the structured JSON
AST representation, not merely the DSL vocabulary, is doing most of the work.

Repair is beneficial but bounded. Compared with direct Verilog,
\texttt{mico\_json\_ast\_repair} has 105 wins, 0 losses, and 81 ties on the
public split, and 30 wins, 0 losses, and 30 ties on held-out. Compared with
plain \texttt{mico\_json\_ast}, repair adds 11 public wins and 2 held-out wins.
Two-sided exact paired tests over discordant attempts give $p<10^{-31}$
against the unstructured public baselines, $p=9.77\times10^{-4}$ against plain
public JSON AST, $p<2\times10^{-9}$ against held-out unstructured baselines,
and no significant held-out difference from plain JSON AST ($p=0.50$).
All accepted repair-turn wins in this run are attributed to the recorded
\texttt{deterministic\_adapter\_instance\_collapse} fallback, so the paper claims
compiler-feedback repair for a common adapter-as-instance error pattern rather
than broad autonomous semantic repair.
