\section{Evaluation: ModuleComposeBench}

ModuleComposeBench evaluates integration of existing RTL/IP modules: wrapper generation, stream chaining, bus attachment, width adaptation, protocol bridging, CDC adapter insertion, and unsafe ambiguity rejection. Metrics include first-pass compose success, lint/elaboration, simulation, bounded formal smoke, unsafe rejection, JSON AST equivalence, QoR delta, and repair turns.

The current artifact answers five evidence-bounded questions: whether intended compositions are accepted, unsafe compositions are rejected, accepted wrappers pass open-source RTL validation, selected ready/valid and width properties survive bounded formal checks, and generated wrappers remain comparable with hand-written references.

\subsection{Deterministic Results}

The public-development manifest has 62 tasks: 36 positives and 26 negatives across L1--L6. A separate held-out manifest adds 12 scoring tasks, including AXI/APB wrapper, video-filter pipeline, explicit CDC event/status composition, and paired negative variants. Benchmark and LLM records include the evaluated manifest path and SHA-256 hash.

The eight subsystem case studies use dedicated RTL under \texttt{rtl/case\_studies/}: T058 streaming accelerator, T059 width/protocol bridge, T060 register/status wrapper, T061 request/response protocol bridge, T062 telemetry chain, and held-out T063--T065 for AXI/APB, video, and CDC event/status composition. All pass the same compiler, JSON AST, lint/elaboration, simulation, selected formal, and QoR gates as the seed tasks. CDC remains an explicit-adapter smoke validation, not a CDC correctness proof.

Simulation covers all 36 positives. Twenty tasks use committed directed testbenches and the remaining 16 use generated ready/valid smoke harnesses from traceability reports. Bounded formal smoke covers 31 single-clock positives: 14 committed directed monitors and 17 generated monitors. Structural and generic-mapped Yosys QoR is available for 9/9 reference-enabled tasks.

A separate host-Vivado subset uses \texttt{scripts/vivado-qor-subset.tcl} through the pinned Windows launcher. It synthesizes measurement-only copies of T001, T003, T058, and held-out T063 for \texttt{xc7a35tcpg236-1}. Generated/reference LUT counts are 16/16, 22/22, 31/31, and 16/16, respectively; WNS is positive for all rows. This is representative out-of-context evidence, not board-level implementation or all-task timing closure.

\subsection{LLM Matrix}

The batch runner supports Direct Verilog, SystemVerilog-interface, MICO source, MICO JSON AST, and JSON AST repair baselines. The current authenticated low-cost matrix ran 62 tasks across two low-cost profiles and five baselines, for 620 attempts. It is a negative result under the current prompts/models.

These data do not support a pass-rate-improvement claim. They do support the artifact contribution: prompts and responses are hashed, compiler/EDA failures are classified, validate-only runs are not counted as failures, and repair turns are represented as auditable JSON patches. A future positive paper requires nonzero structured positive passes and paired comparisons across at least two profiles; otherwise the correct framing is a compiler-gated benchmark/tool paper plus negative LLM study.
