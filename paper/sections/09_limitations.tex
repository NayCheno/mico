\section{Limitations and Threats to Validity}

MICO requires interface and adapter libraries, and an overly restrictive type system can reduce useful model search space. The current v0 contract checker is a ready/valid subset, not arbitrary LTL. The 83 public-development tasks, 40-task held-out split, and 30-task deterministic realism supplement provide useful coverage but do not by themselves support broad industrial generalization; \texttt{docs/release\_claim\_table.md} records the evidence mapping for these numbers. Simulation covers all positives with committed directed harnesses. Formal smoke covers 40 public single-clock positives with committed directed monitors, the held-out split has 17 committed single-clock formal monitors, and the supplemental realism manifest has 13 committed single-clock formal monitors. These are bounded ready/valid and payload checks, not exhaustive formal signoff; CDC proof remains unclaimed.

QoR evidence is limited to structural area/wire counts, generic mapped-cell counts against small references, and a 12-task Vivado out-of-context measurement subset. It is not routed timing closure, technology-mapped delay for the full benchmark, or board-level implementation. The authenticated LLM result is positive for the tested OpenCode Go profiles and prompts, but it is not a claim about arbitrary models, future provider versions, or unbounded repair. Recorded repair wins currently come from a deterministic fallback for one adapter-instance error pattern. Provider results may vary across model versions and API settings, so prompts, model identifiers, hashes, compiler/EDA results, costs, and generated artifacts must be logged.
