\begin{abstract}
Large language models can generate small Verilog fragments, but real RTL integration remains brittle because correctness depends on port binding, direction, width, handshake protocol, clock/reset domains, CDC/RDC handling, wrappers, and adapter selection. This paper presents MICO, a module-interface-contract-oriented hardware front-end language and Rust compiler framework. Rather than asking an LLM to emit final RTL directly, MICO restricts model output to interface schemas, module instance graphs, connection topology, adapter plans, and contract skeletons. A deterministic compiler then performs name resolution, interface compatibility checks, protocol/domain checks, adapter legality validation, and emits traceable SystemVerilog wrappers, SVA collateral, and future CIRCT HW/ESI/Verif/LTL IR. The paper defines the problem, language core, compiler architecture, and ModuleComposeBench evaluation plan, arguing that LLM-assisted hardware design should move from free-form RTL text generation toward compiler-checked module composition.
\end{abstract}
