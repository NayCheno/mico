\section{Adapter Synthesis}

Adapters are explicit first-class artifacts. Width adapters, protocol bridges, pipeline stages, buffers, and CDC FIFOs must appear in the typed IR and elaboration report. A clock-domain crossing is illegal by default unless the graph declares a CDC adapter. Protocol adaptation is illegal unless an adapter in the library states the source and destination contracts it preserves.

This design is especially important for LLM-assisted flows. If the model silently inserts glue logic, the result may compile while changing latency, backpressure, ordering, or reset semantics. MICO forces these choices into named objects that can be audited, tested, and benchmarked.

Adapter legality is defined at the interface boundary. A width adapter states source/destination payload widths and whether the transformation is extension, narrowing with proof, packing, or unpacking. A CDC adapter states source/destination clock-reset domains and synchronization mechanism. A protocol bridge states which ordering, liveness, and backpressure properties it preserves.

The current implementation supports explicit width and CDC adapter declarations for benchmark tasks, parses a small ready/valid contract subset, and checks known adapter guarantees against sink-side requirements. Generated artifacts include SystemVerilog wrappers, traceability reports, and basic SVA skeletons. Directed formal collateral covers selected width and stream properties; CDC collateral remains smoke-only.
