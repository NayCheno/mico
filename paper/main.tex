\begin{abstract}
Large language models can generate small Verilog fragments, but real RTL integration remains brittle because correctness depends on port binding, direction, width, handshake protocol, clock/reset domains, CDC/RDC handling, wrappers, and adapter selection. This paper presents MICO, a module-interface-contract-oriented hardware front-end language and Rust compiler framework for LLM-assisted module composition. Rather than asking an LLM to emit final RTL directly, MICO restricts model output to interface schemas, module instance graphs, connection topology, adapter plans, and contract skeletons. A deterministic compiler then performs name resolution, interface compatibility checks, protocol/domain checks, adapter legality validation, and emits traceable SystemVerilog wrappers, SVA skeletons, and future CIRCT HW/ESI/Verif/LTL IR. We define the module-composition problem, the MICO language core, compiler passes, LLM repair loop, adapter discipline, and ModuleComposeBench evaluation protocol. The current open-source artifact validates the deterministic end-to-end path on 62 tasks, including five dedicated subsystem case studies, with Rust unit tests, machine-readable diagnostics, JSON AST and JSON IR paths, a ready/valid v0 contract subset, traceability reports, 36 positive lint/elaboration checks, 26 unsafe-rejection tasks, Verilator/Icarus/Yosys checks, 36 positive Icarus smoke simulations, 31 single-clock bounded SymbiYosys smoke proofs, structural and generic-mapped Yosys QoR summaries, and an authenticated low-cost LLM matrix that currently yields negative results. It does not yet claim full per-task formal coverage, timing QoR, arbitrary LTL, or LLM pass-rate improvements. The central argument is that LLM-assisted hardware design should move from free-form RTL text generation toward compiler-checked, reproducible module composition.
\end{abstract}
