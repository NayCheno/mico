\begin{abstract}
LLMs can generate small Verilog fragments, but RTL integration remains brittle because correctness depends on port binding, direction, width, handshake protocol, clock/reset domains, CDC/RDC handling, adapters, and wrappers. MICO reframes LLM-assisted integration as compiler-checked module composition: models propose typed interface graphs and repair patches, while a Rust compiler validates the graph and emits traceable SystemVerilog, SVA skeletons, JSON IR, and diagnostics. The artifact currently passes 62/62 deterministic ModuleComposeBench tasks, including 36/36 positive compositions, 26/26 unsafe rejections, 36/36 simulations, 31/31 single-clock bounded formal smoke checks, 9/9 structural/generic Yosys QoR rows, a four-task Vivado out-of-context subset, and a held-out case-study split. An authenticated low-cost LLM matrix is negative: current prompts/models do not produce positive-task compiler or lint passes. The paper therefore claims a benchmark, compiler gate, repairable representation, and failure-taxonomy workflow, not full formal coverage, timing closure, arbitrary LTL, or LLM pass-rate improvement.
\end{abstract}
