\section{Rust Compiler Framework}

MICO is implemented as a Rust workspace with four initial crates:

\begin{itemize}
  \item \texttt{mico\_ir}: AST, typed IR, diagnostics, and semantic checks.
  \item \texttt{mico\_frontend}: lexer, parser, source maps, and recovery.
  \item \texttt{mico\_codegen}: JSON IR, SystemVerilog, and SVA emission.
  \item \texttt{mico\_cli}: \texttt{check}, \texttt{dump-ir}, \texttt{emit-sv}, and future benchmark commands.
\end{itemize}

The compiler pipeline is deterministic:

\begin{enumerate}
  \item The frontend lexes and parses MICO source into an AST with source locations.
  \item The IR layer resolves declarations, instances, endpoints, domains, adapter kinds, and contract metadata.
  \item The checker rejects duplicate declarations, unknown endpoints, illegal directions, unsupported adapter kinds, width mismatches, domain mismatches, and missing contract obligations.
  \item Code generation emits JSON IR, SystemVerilog wrappers, and SVA skeletons from checked IR only.
  \item The CLI exposes \texttt{parse}, \texttt{check}, \texttt{build}, \texttt{dump-ir}, \texttt{emit-sv}, \texttt{emit-sva}, \texttt{verify}, and \texttt{report}.
\end{enumerate}

The first implementation stage intentionally avoids a full MLIR binding. It emits conservative SystemVerilog wrappers, structured JSON IR, and SVA skeletons. A later stage can lower the typed IR into CIRCT HW modules, ESI channels, and Verif/LTL assertions. Rust is a pragmatic implementation language for this front end because its enums, pattern matching, ownership model, and test ecosystem fit compiler passes and diagnostics well.
