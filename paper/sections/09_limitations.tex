\section{Limitations and Threats to Validity}

MICO requires interface and adapter libraries, and an overly restrictive type system can reduce useful model search space. The current v0 contract checker is a ready/valid subset, not arbitrary LTL. The 62 public-development tasks and 20-task held-out split provide useful coverage but do not by themselves support broad industrial generalization. Simulation covers all positives; in the public-development split, 32 positives use task-specific directed harnesses and 4 use generated ready/valid smoke harnesses. Formal smoke covers 31 public single-clock positives, with 24 directed monitors and 7 generated monitors. The held-out split now has directed formal monitors for the non-CDC subsystem cases, but CDC proof remains unclaimed.

QoR evidence is limited to structural area/wire counts, generic mapped-cell counts against small references, and a nine-task Vivado out-of-context measurement subset. It is not routed timing closure, technology-mapped delay for the full benchmark, or board-level implementation. The authenticated LLM result is positive for the tested OpenCode Go profiles and prompts, but it is not a claim about arbitrary models, future provider versions, or unbounded repair. Accepted repair-turn wins currently come from a recorded deterministic fallback for one adapter-instance error pattern. Provider results may vary across model versions and API settings, so prompts, model identifiers, hashes, compiler/EDA results, costs, and generated artifacts must be logged.
