\section{Motivation: Connection Entropy}

We use connection entropy to describe amplification from semantic intent to primitive RTL wiring:

\begin{equation}
  CE = \frac{\text{primitive signal edges}}{\text{semantic interface edges}}.
\end{equation}

A simple producer-FIFO-consumer chain may be one semantic edge sequence, but it expands into payload, valid, ready, reset, and instance-specific port bindings. As connection entropy grows, LLMs omit ports, reverse ready/valid direction, mismatch widths, confuse reset polarity, cross clock domains without synchronization, or create wrappers that pass weak tests for the wrong reason. Prompt engineering can reduce these failures but does not remove the coupled obligations from the target string.

For example, direct RTL prompts in the matrix often produce a wrapper that names
the intended producer, FIFO, and consumer but leaves ready/valid payload bits
unconnected or drives a sink-side status port from the wrong direction. A MICO
JSON AST answer for the same request names the instances and interface edges
instead. The compiler can then reject a reversed edge before code generation or
accept the graph and emit the concrete port bindings, traceability JSON, and
directed simulation wrapper from the checked IR. The improvement is therefore
not that the model writes better Verilog text; it is that the acceptance target
is smaller, typed, and auditable.

MICO constrains the output space:

\begin{itemize}
  \item LLMs produce candidates, not trusted RTL.
  \item Direction, width, protocol, and domain mismatches are compile-time errors.
  \item CDC and RDC crossings require explicit adapters.
  \item Automatically inserted buffers, pipelines, width adapters, or CDC FIFOs must appear in an elaboration report.
  \item Verification collateral is generated from interface contracts.
\end{itemize}

This distinguishes local RTL complexity from composition complexity. MICO does not try to synthesize arbitrary datapaths; it checks whether already-existing modules are connected safely before RTL tools elaborate the wrapper.
