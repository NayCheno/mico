\section{Limitations and Threats to Validity}

MICO requires an interface library and adapter library, both of which need engineering effort. A type system that is too restrictive may reduce the useful search space available to the model. CIRCT lowering also introduces integration cost. Finally, ModuleComposeBench must include realistic module-composition tasks; otherwise the benchmark could collapse into a schema-generation exercise.

These risks are manageable if the project remains narrowly scoped. The first target is not a universal HDL. It is a compiler-checked interconnect layer for LLM-assisted RTL integration.

There are also evaluation threats. Twelve seed tasks are too small to support statistical claims. Verilator, Icarus, Yosys, the positive-seed simulations, selected bounded SymbiYosys proofs, structural Yosys QoR summaries, and the minimal SymbiYosys smoke proof catch syntax, elaboration, tool-path, and simple structural issues, but they do not prove protocol correctness or timing closure for each benchmark task. The ready/valid contract checker is a v0 subset, not arbitrary LTL. Simulation is currently limited to the four positive seed tasks, formal proof is limited to the direct and width seed properties, and QoR is limited to structural area/wire counts against small hand-written references. LLM provider results may vary across model versions and API settings; therefore prompts, model identifiers, profile settings, output hashes, compiler results, EDA results, cost metadata, and generated artifacts must be logged. The repository template intentionally leaves model price fields unset, so cost estimates require a local ignored configuration. Finally, a benchmark built only from hand-written toy modules would overstate usefulness. The full evaluation must include realistic IP wrappers, negative safety tasks, and transparent reporting of failed repairs.

Artifact reproducibility has one practical limitation in this revision: the Docker EDA image used for Rust, Python, RTL tools, benchmark scoring, and LLM validation does not include LaTeX. The paper PDF build therefore depends on the documented Windows-host TeX Live toolchain unless LaTeX is added to the Docker image in a future artifact revision.

MICO's safety argument is consequently layered, not absolute. The compiler should prevent known illegal compositions early; RTL tools should reject invalid generated wrappers; simulation and formal harnesses should validate behavior; and human review should inspect adapter libraries, waivers, and benchmark construction.
