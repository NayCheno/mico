\section{Limitations and Threats to Validity}

MICO requires interface and adapter libraries, and an overly restrictive type system can reduce useful model search space. The current v0 contract checker is a ready/valid subset, not arbitrary LTL. The 83 expanded deterministic public-development tasks, 40-task expanded deterministic held-out split, and 30-task deterministic realism supplement provide useful coverage but do not by themselves support broad industrial generalization; \texttt{docs/release\_claim\_table.md} records the evidence mapping for these numbers, and \texttt{docs/submission\_claim\_lock\_2026Q3.md} prevents mixing these denominators with locked pre-expansion LLM numerators. Simulation covers all positives with committed directed harnesses. Formal smoke covers 40 public single-clock positives with committed directed monitors, the held-out split has 17 committed single-clock formal monitors, and the supplemental realism manifest has 13 committed single-clock formal monitors. These are bounded single-clock proof obligations for ready/valid stability, fire-transfer, self-loop, no-drop/no-duplicate, width, register/status, protocol, and telemetry relations, not exhaustive formal signoff. CDC rows are structural adapter-boundary and direct-rejection checks only; CDC correctness proof remains unclaimed.

QoR evidence is limited to structural area/wire counts, generic mapped-cell counts against small references, and a 12-task Vivado out-of-context measurement subset. It is not routed timing closure, technology-mapped delay for the full benchmark, or board-level implementation. The authenticated LLM result is positive for the tested OpenCode Go profiles and prompts, but it is not a claim about arbitrary models, future provider versions, or unbounded repair. Recorded repair wins currently come from a deterministic fallback for one adapter-instance error pattern. Provider results may vary across model versions and API settings, so prompts, model identifiers, hashes, compiler/EDA results, costs, and generated artifacts must be logged.

Source spans are strongest for \texttt{.mico} input. JSON AST input preserves graph-node identities for diagnostics and repair, but source spans may be null; in that path, the \texttt{nodes} arrays and traceability \texttt{source\_ref} records are the stable fallback for localization.
